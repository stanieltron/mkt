# Makeit Protocol Monorepo

Full-stack DeFi protocol — Solidity contracts, Rust backend, React frontend.

## Repo Structure

```
backend/              Rust API, chain sync, liquidation bot, Redis cache, WS gateway
frontend/             User/admin app (Vite + React)
solidity/             Main protocol contracts + tests (Foundry)
local_deploy_rust/    Local-only deploy contracts and test infrastructure
scripts/              Root orchestration scripts
scenarios/            Simulation and scenario artifacts
e2e/                  End-to-end test scripts
```

Auto-generated or gitignored directories (safe to delete and regenerate):

```
foundry/              Foundry tooling — binaries + forge-std (run npm run install:foundry)
cache/                Foundry build cache
target_temp/          Rust build output (Windows local dev)
```

## Prerequisites

- Node.js 20+
- Rust (stable) + Cargo
- Docker (for local Postgres + Redis)

> `npm run install:foundry` first uses global Foundry (`forge/anvil/cast/chisel`) if available, and only downloads local binaries when missing.

## Workflows

Every command is self-contained. Clone the repo and run directly.

### First time / install everything

```bash
npm run install:all
```

Installs: Foundry tooling (global if present, otherwise local fallback) + forge-std, root npm packages, frontend npm packages, Rust dependencies.

### Solidity tests

```bash
npm run test:solidity
```

Auto-installs Foundry if not present, then runs the full test suite.

### Frontend development

```bash
npm run dev:fe
```

Installs frontend deps if needed, then starts the Vite dev server.

### Backend development

```bash
npm run dev:be
```

Starts the Rust backend (assumes local env files already generated by `npm run local`).

### Full local stack (Anvil + deploy + backend + frontend)

```bash
npm run local
```

Fresh start: installs everything, starts Anvil, deploys all contracts, generates env files, starts Postgres + Redis, starts backend + frontend.

What it does internally (`scripts/dev-rust.js`):

1. Starts Anvil (`scripts/anvil.js`)
2. Deploys local contracts (`scripts/deploy-local.js`)
3. Writes `local_deploy_rust/.env` and `e2e/.env`
4. Starts Postgres + Redis (`docker-compose.yml`)
5. Runs DB migrations + backend init
6. Starts Rust backend (hot reload via `cargo-watch` if available)
7. Starts frontend dev server

### Split local stack (run components separately in multiple shells)

Use this when you want to restart frontend/backend independently without stopping the whole stack.

#### 1) Local chain only (Anvil + contract deploy + local env generation)

```bash
npm run local:chain
```

Fresh reset of chain + redeploy:

```bash
npm run local:chain:fresh
```

This generates/updates:
- `local_deploy_rust/.env`
- `e2e/.env`

#### 2) Backend modes

Non-local backend (uses your current shell/root `.env` values):

```bash
npm run run:be
```

Local backend mode (loads `local_deploy_rust/.env`, starts DB/Redis + migrations, runs Rust backend + local relay with swap runner):

```bash
npm run run:be:local
```

Fresh local DB reset:

```bash
npm run run:be:local:fresh
```

#### 3) Frontend modes

Non-local frontend (uses normal frontend env/config):

```bash
npm run run:fe
```

Local frontend mode (loads `local_deploy_rust/.env`, enables local Vite config, faucet/runner integration):

```bash
npm run run:fe:local
```

### Frontend tests

```bash
npm run test:fe
```

### Backend tests

```bash
npm run test:be
```

### E2E tests

```bash
npm run test:e2e
```

Auto-installs everything, then runs the full E2E scenario against a fresh local stack.

### All tests

```bash
npm run test
```

Runs: `test:solidity` → `test:fe` → `test:be`

### Production build + start

```bash
npm run build    # install + build backend release + build frontend bundle
npm start        # start backend (no DB reset)
npm run start:fresh  # reset local DB + migrate + start
```

## Environment Files

- `.env.example` — **committed template** for production config. Copy to `.env` and fill in values.
- `.env` — your real secrets. **Never committed** (gitignored).
- `local_deploy_rust/.env` — **auto-generated** by `npm run local`. Contains local chain addresses, wallet keys, runtime config.
- `e2e/.env` — **auto-generated** by `npm run local`. Contains only values needed by the E2E scripts.

## Railway Deployment

### Backend service

- Root directory: repo root
- Build command: `npm run build`
- Start command: `npm start`

Required env vars: `RPC_URL`, `CHAIN_ID`, `DATABASE_URL`, `REDIS_URL`, `MAKEIT_ADDRESS`, `ORACLE_ADDRESS`, `SWAP_ADAPTER_ADDRESS`, `UNISWAP_POOL_ADDRESS`, `ADMIN_USERNAME`, `ADMIN_PASSWORD`

Provision in Railway: Postgres plugin + Redis plugin.

### Frontend service

Set root directory to `frontend/`:

- Build command: `npm run build`
- Start command: `npm run start`

Required env vars: `BACKEND_URL`, contract addresses, token addresses.

## Foundry / Solidity Notes

Foundry tooling lives in `foundry/` (gitignored). On a fresh clone:

```bash
npm run install:foundry   # uses global Foundry if present; otherwise downloads local binaries; then clones forge-std
```

This is called automatically by `test:solidity`, `local`, `install:all`, and `test:e2e`.

To override the binary location: set `FORGE_BIN` or `ANVIL_BIN` env vars.

## Architecture Notes

- Blockchain is source of truth for all trade lifecycle state
- Backend chain-sync worker indexes `TradeOpened` / `TradeClosed` events
- Redis is hot read cache for main-page data
- WebSocket endpoint `/ws?wallet=0x...` for live price + trade updates
- Frontend uses bootstrap snapshot + WS stream (no tight polling)

## Troubleshooting

| Problem | Fix |
|---|---|
| `forge not found` | `npm run install:foundry` |
| Backend config errors at startup | Check root `.env` has all required vars, or run `npm run local` to regenerate local env |
| Frontend can't reach backend | Check `BACKEND_URL` and CORS/network routing |
| Local DB/Redis issues | `npm run local` (fresh restart) |
| Solidity build errors | `npm run install:foundry` to ensure forge-std is present |
